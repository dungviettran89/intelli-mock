# Task: CLI E2E Tests

**Status:** completed
**Created:** 2026-04-11
**Priority:** medium
**Estimated Time:** 20-30 mins

## Context
The CLI application now has a working `start` command, but has no test coverage. We need E2E tests that validate the full user journey through the CLI entry point to ensure it works correctly in real-world usage.

## Scope
### In Scope
- [ ] Create `apps/intelli-mock/test/` directory structure
- [ ] Implement `cli-runner.ts` helper to spawn CLI and capture output
- [ ] Create vitest config for CLI E2E tests
- [ ] Write E2E tests for CLI start command
- [ ] Write E2E tests for auth flags
- [ ] Write E2E tests for UI serving
- [ ] Add test scripts to CLI package.json
- [ ] Update root vitest.workspace.ts to include CLI tests

### Out of Scope
- Unit tests for CLI config loader (separate task)
- Playwright UI E2E tests (separate task)
- Integration tests for core (already exist)

## Steps
1. Create test directory structure: `apps/intelli-mock/test/helpers/` and `apps/intelli-mock/test/e2e/`
2. Add vitest as devDependency to CLI package.json
3. Create `test/helpers/cli-runner.ts` - CLI process spawner
4. Create `test/vitest.config.ts` - Vitest configuration
5. Create `test/e2e/cli-start.test.ts` - Basic start/shutdown tests
6. Create `test/e2e/cli-auth-flags.test.ts` - Auth flag validation
7. Add `test` and `test:e2e` scripts to CLI package.json
8. Update root `vitest.workspace.ts` to include CLI test config
9. Run tests and verify they pass
10. Update PROGRESS.md

## Acceptance Criteria

- [x] Tests validate auth flag combinations
- [x] All tests complete in < 30 seconds (9.51s actual)
