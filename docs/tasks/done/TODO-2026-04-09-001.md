## Context
Phase 1 of the architecture requires TypeORM with dual DB support (sql.js for dev, MariaDB for prod) and all 6 entity models. Currently no database layer exists in the codebase. This is foundational — all services and APIs depend on it.

## Scope
### In Scope
- [x] Install TypeORM + sql.js + mariadb dependencies in @intelli-mock/core
- [x] Create cross-driver compatibility layer (`packages/intelli-mock-core/src/config/database.ts`)
- [x] Create TypeORM DataSource factory (`packages/intelli-mock-core/src/database/data-source.ts`)
- [x] Create all 6 entity models:
  - Tenant entity
  - MockEndpoint entity
  - SamplePair entity
  - MockScript entity
  - TrafficLog entity
  - User entity
- [x] Export entities from `packages/intelli-mock-core/src/index.ts`
- [x] Initialize DataSource on app startup

### Out of Scope
- Database migrations
- Service implementations
- JWT auth middleware
- tsyringe container setup (already partially done)
- CLI `start` / `init` commands

## Steps
1. Add TypeORM, sql.js, mariadb dependencies to @intelli-mock/core package.json
2. Create `config/database.ts` — environment-driven TypeORM config with driver compatibility
3. Create `database/data-source.ts` — DataSource factory that reads env vars
4. Create all 6 entity files under `entities/` with proper TypeORM decorators and relationships
5. Update `app.ts` to initialize TypeORM on startup
6. Update `index.ts` to export entities and database module
7. Run build and typecheck to verify
