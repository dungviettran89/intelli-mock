# Task: Set Up tsyringe DI Container + JWT Auth Middleware

**Status:** completed
**Created:** 2026-04-09
**Completed:** 2026-04-09
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Phase 1 of the architecture requires a tsyringe DI container and JWT auth middleware with tenant resolution. The entity models and TypeORM are already set up (TODO-2026-04-09-001 completed), but there's no dependency injection or authentication layer. All future services and controllers will depend on these foundations.

**JWT Strategy:** Asymmetric (RS256/ES256) — a public key (or certificate) is configured in Intelli-Mock to verify tokens signed by a private key held by the upstream identity provider / proxy gateway.

## Scope
### In Scope
- [ ] Create `packages/intelli-mock-core/src/container.ts` — tsyringe root container with service registrations
- [ ] Create `packages/intelli-mock-core/src/config/env.ts` — centralized environment variables module (Zod or manual validation)
- [ ] Create `packages/intelli-mock-core/src/core/auth/jwt.middleware.ts` — JWT verification middleware extracting tenant claim
- [ ] Create `packages/intelli-mock-core/src/core/auth/user-resolver.ts` — TenantResolver service (upsert tenant by slug, upsert user by sub)
- [ ] Wire container into `app.ts`
- [ ] Update `packages/intelli-mock-core/src/index.ts` to export new modules

### Out of Scope
- Route matcher (longest-match logic)
- Mock/Sample/Script services and controllers
- Database migrations
- CLI commands (start/init)

## Steps
1. Create `config/env.ts` — centralized env var reading with defaults and validation (PORT, DB_TYPE, **JWT_PUBLIC_KEY** (PEM string or file path), JWT_ISSUER, AI_PROVIDER, AI_BASE_URL, AI_API_KEY, AI_MODEL)
2. Create `container.ts` — tsyringe root container, register DataSource, register upcoming services as singletons/transients
3. Create `core/auth/jwt.middleware.ts` — Express middleware that verifies JWT using the **public key** (RS256/ES256), extracts `tenant` claim, calls TenantResolver, attaches resolved tenant/user to `req`
4. Create `core/auth/user-resolver.ts` — service that upserts Tenant by slug and User by sub in a single request pipeline
5. Wire container initialization into `app.ts` (call after DataSource init)
6. Update `index.ts` exports
7. Run `pnpm -r build` and `pnpm -r typecheck` to verify

## Acceptance Criteria
- [ ] `pnpm -r build` compiles without errors
- [ ] `pnpm -r typecheck` passes with no type errors
- [ ] `container.ts` exports a configured Container instance with registered dependencies
- [ ] JWT middleware correctly rejects requests with missing/invalid tokens (401)
- [ ] JWT middleware correctly rejects unrecognized tenants (403)
- [ ] TenantResolver upserts Tenant and User records on valid JWT
- [ ] Resolved tenant/user attached to `req` for downstream middleware/controllers
- [ ] `env.ts` validates required env vars on startup
