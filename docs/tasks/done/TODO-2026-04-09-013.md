# Task: E2E Integration Test — External API + Ollama

**Status:** completed  
**Completed:** 2026-04-09  
**Created:** 2026-04-09  
**Priority:** high  
**Estimated Time:** 20-30 mins  

## Context

The project has a comprehensive unit testing architecture (27 test files, 212 tests, 100% offline), but lacks **integration tests** that validate the full system working together with real external services.

This task implements a **manual integration test** that:
1. Makes real API calls to **JSONPlaceholder** (`https://jsonplaceholder.typicode.com`)
2. Calls a real local **Ollama** instance with **`gemma4:31b-cloud`** model
3. Validates that Intelli-Mock can proxy traffic, capture samples, generate AI scripts, and serve mocks

This test is **NOT part of CI** — it's designed for developer validation, debugging, and demos.

**Design doc:** `docs/INTEGRATION-TEST-DESIGN.md`

## Scope

### In Scope
- [ ] Directory structure: `test/integration/` with helpers, scenarios, and runner
- [ ] Health check helpers: Ollama connectivity + model verification, JSONPlaceholder reachability
- [ ] Test server setup: Start Intelli-Mock with in-memory sql.js database
- [ ] Main runner script: `run-integration.ts` orchestrated via npm
- [ ] E2E scenario: Proxy → Capture → AI Generate → Mock Intercept → Validate
- [ ] Ollama-only scenario: Isolated script generation validation
- [ ] Test report output: Formatted success/failure summary with timing metrics
- [ ] Package.json scripts: `test:integration` at root and core package level
- [ ] Vitest integration config: Separate config with longer timeouts
- [ ] README.md: Setup instructions and prerequisites

### Out of Scope
- Proxy module implementation (doesn't exist yet — test will use existing mock handler + traffic capture)
- CI/CD integration (manual-only for now)
- Playwright/browser testing
- Multiple external APIs (JSONPlaceholder only for this task)
- Auto-endpoint (`/_it/auto/*`) — only `/_it/mock/*` for now

## Steps

1. **Create directory structure**
   ```
   packages/intelli-mock-core/test/integration/
   ├── README.md
   ├── run-integration.ts
   ├── vitest.integration.config.ts
   └── helpers/
   │   ├── ollama-health.ts
   │   ├── external-api.ts
   │   ├── test-server.ts
   │   └── report.ts
   └── scenarios/
       ├── e2e-proxy-to-mock.test.ts
       └── ollama-generation.test.ts
   ```

2. **Implement health check helpers**
   - `helpers/ollama-health.ts`: Verify Ollama is running at `http://localhost:11434`, check `gemma4:31b-cloud` model is available
   - `helpers/external-api.ts`: Verify JSONPlaceholder is accessible, validate response structure

3. **Implement test server setup**
   - `helpers/test-server.ts`: Start Intelli-Mock Express app with sql.js in-memory DB, listen on random port, return server instance + cleanup function

4. **Implement report formatter**
   - `helpers/report.ts`: Track test steps, timing, success/failure, output formatted table

5. **Implement main runner script**
   - `run-integration.ts`: Parse CLI args, run health checks, execute scenarios sequentially, print report, exit with code

6. **Implement E2E scenario**
   - `scenarios/e2e-proxy-to-mock.test.ts`:
     - Create mock endpoint with `proxyUrl` to JSONPlaceholder
     - Send GET `/api/posts/1` through Intelli-Mock
     - Verify real response returned (status 200, has userId/id/title)
     - Trigger AI generation with captured samples → Ollama generates script
     - Validate generated script syntax
     - Send GET `/api/posts/1` again
     - Verify mock intercepts (faster response, traffic log shows mock source)

7. **Implement Ollama-only scenario**
   - `scenarios/ollama-generation.test.ts`:
     - Direct call to AIService with sample pairs
     - Validate generated script structure (has `module.exports`, `async`)
     - Validate model is `gemma4:31b-cloud`
     - Validate token usage > 0

8. **Add package.json scripts**
   - Root: `"test:integration": "pnpm --filter @intelli-mock/core test:integration"`
   - Core: `"test:integration": "node --loader ts-node/esm test/integration/run-integration.ts"`

9. **Add Vitest integration config**
   - `test/integration/vitest.integration.config.ts`: 60s timeout, include scenarios only

10. **Write README.md**
    - Prerequisites (Ollama, gemma4:31b-cloud model)
    - Setup instructions
    - How to run: `pnpm test:integration`
    - Environment variables
    - Troubleshooting
