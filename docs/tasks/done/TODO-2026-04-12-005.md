# Task: CLI `init` Command

**Status:** completed
**Created:** 2026-04-12
**Completed:** 2026-04-12
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
The CLI application currently only has the `start` command. The `init` command is a planned feature from the CLI Application section in PROGRESS.md. It should scaffold a basic configuration file for users, making it easier to get started with Intelli-Mock without manually setting environment variables.

This task aligns with Functional Requirement #20: "CLI application with `start` and `init` commands".

## Scope
### In Scope
- [ ] Implement `init` command in `apps/intelli-mock/src/commands/init.ts`
- [ ] Add command registration in `apps/intelli-mock/src/cli.ts`
- [ ] Generate default config file (JSON or YAML) with sensible defaults
- [ ] Support interactive prompts for key configuration (optional, can use defaults)
- [ ] Add unit tests for `init` command

### Out of Scope
- Advanced configuration options (can be added later)
- Migration of existing config files
- Validation of external resources (e.g., checking if AI endpoint is reachable)

## Steps
1. Create `apps/intelli-mock/src/commands/init.ts` with Commander command definition
2. Implement config file generation logic:
   - Check if config file already exists (warn if yes)
   - Generate default config with sensible defaults (db type, ports, AI endpoint)
   - Write to `.intelli-mock.json` or `intelli-mock.config.json`
   - Support `--output` flag to specify custom path
3. Register command in `apps/intelli-mock/src/cli.ts`
4. Write unit tests in `apps/intelli-mock/test/commands/init.test.ts`:
   - Test default config generation
   - Test custom output path
   - Test existing file warning behavior
   - Test file content validation
5. Run tests and verify command works end-to-end

## Acceptance Criteria
- [x] `intelli-mock init` command generates a valid config file with defaults
- [x] `intelli-mock init --output custom.json` writes to specified path
- [x] Command warns when config file already exists (doesn't overwrite without flag)
- [x] `intelli-mock init --force` overwrites existing config
- [x] At least 4 unit tests covering init command behavior (8 tests implemented)
- [x] Command integrates with existing CLI structure (follows same patterns as `start` command)
- [x] Generated config file is compatible with existing config loader (`apps/intelli-mock/src/config.ts`)
