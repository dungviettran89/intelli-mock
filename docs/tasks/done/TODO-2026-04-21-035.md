# Task: Implement App Shell & Global Layout

**Status:** completed
**Created:** 2026-04-21
**Priority:** high
**Estimated Time:** 30 mins

## Context
The individual UI components are currently rendered without a global navigation structure. This task implements the App Shell as defined in `docs/UI.md` and the `mock-endpoints.html` mockup, providing a unified sidebar and header for the application.

## Scope
### In Scope
- [x] Create `packages/intelli-mock-ui/src/components/app-shell.ts`.
- [x] Implement the **Sidebar** with navigation links for Mocks, Traffic, and Settings.
- [x] Implement the **Top Header** with breadcrumbs, tenant indicator, and user profile.
- [x] Implement basic **state-based routing** to switch between `<mock-list>`, `<traffic-log-viewer>` (placeholder), and `<settings-panel>`.
- [x] Update `packages/intelli-mock-ui/index.html` to use `<app-shell>` as the root component.
- [x] Ensure "Technical Editorial" styling (Red/Black/White theme) is applied globally via the shell.

### Out of Scope
- Implementation of the actual Traffic Log Viewer (this remains a placeholder).
- Complex URL-based routing (state-based is sufficient for now).

## Steps
1. Create `app-shell.ts` and define the layout structure using Lit Element and Vanilla CSS.
2. Define the navigation state and a helper to render the active component in the main content slot.
3. Apply the branding styles: Sidebar (Background: #303030, Brand: #D32F2F), Header (Glassmorphism), and Typography (Inter/Space Grotesk).
4. Register the component in `index.ts`.
5. Update `index.html` to include Google Fonts (Inter, Space Grotesk) and Material Symbols, and mount `<app-shell>`.
6. Refactor `mock-list.ts` to remove its internal "Intelli-Mock" header so it fits cleanly into the shell.
