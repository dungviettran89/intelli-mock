## Context
The MockService (CRUD, tenant-scoped) and RouteMatcher (longest-match routing) are implemented but have no HTTP entry points. This task builds the REST API layer — Express controller + router — so users can create, list, read, update, and delete mock endpoints via standard HTTP.

The controller extracts `tenantId` from `req.tenant.id` (set by JWT middleware), delegates to MockService, and returns JSON responses with proper status codes.

## Scope
### In Scope
- [ ] `src/modules/mock/mock.controller.ts` — REST handlers (POST, GET, GET/:id, PUT/:id, DELETE/:id)
- [ ] `src/modules/mock/mock.routes.ts` — Express Router mounted at `/api/mocks`
- [ ] Wire routes into `createApp()` in `src/app.ts`
- [ ] `test/modules/mock/mock.controller.test.ts` — HTTP-level tests (supertest, fully mocked)
- [ ] `test/modules/mock/mock.integration.test.ts` — integration tests via test app (controller + service + matcher)

### Out of Scope

## Steps
1. **Create MockController** — class with methods for each CRUD operation, injecting MockService and RouteMatcher via tsyringe
2. **Create mock.routes.ts** — Express Router with proper middleware chain, mounted at `/api/mocks`
3. **Wire into app.ts** — import and register routes after auth middleware
4. **Write controller tests** — supertest-based HTTP tests with mocked service layer
5. **Write integration tests** — test app with real service (mocked DB), verify end-to-end flow
6. **Validate** — build, typecheck, test

## Acceptance Criteria
- [ ] POST `/api/mocks` creates endpoint, returns 201 with body
- [ ] GET `/api/mocks` lists all endpoints for tenant, returns 200 with array
- [ ] GET `/api/mocks/:id` returns single endpoint or 404
- [ ] PUT `/api/mocks/:id` updates endpoint, returns 200
- [ ] DELETE `/api/mocks/:id` deletes endpoint, returns 204
- [ ] All responses include proper error codes (400 bad request, 404 not found, 500 server error)
- [ ] All queries scoped by tenantId from req.tenant
- [ ] All tests pass (100% offline, fully mocked)
