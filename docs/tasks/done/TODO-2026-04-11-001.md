## Steps
1. Create `AutoHandler` class in `packages/intelli-mock-core/src/modules/mock/auto.handler.ts`
   - Constructor injects ProxyService, MockService, RouteMatcher, TrafficService
   - `handleRequest(req, res)` method implementing the auto-endpoint pipeline:
     1. Find matching MockEndpoint via RouteMatcher
     2. Check if `proxy_url` is configured
     3. If yes: forward request via ProxyService
        - Success → log traffic → return response
        - Failure → log error → fall back to mock execution
     4. If no proxy_url: fall back to mock execution
     5. Execute mock script via ScriptRunner (same as mock handler)
     6. Log traffic → return response
2. Register auto-handler route in `packages/intelli-mock-core/src/modules/mock/mock.routes.ts`
   - Add `router.all('/auto/*', ...)` route
3. Write comprehensive unit tests in `packages/intelli-mock-core/test/modules/mock/auto.handler.test.ts`
   - Test cases:
     - Proxy success: forwards request, returns response, logs traffic
     - Proxy failure: catches error, falls back to mock, logs error
     - No proxy_url: skips proxy, executes mock directly
     - Mock unavailable: returns 502 when no active script
     - Endpoint not found: returns 404
   - Mock ProxyService, MockService, RouteMatcher, ScriptRunner, TrafficService
4. Add integration test in `packages/intelli-mock-core/test/modules/mock/auto.handler.integration.test.ts`
   - Test full request/response cycle with mocked dependencies
5. Update `src/index.ts` exports to include AutoHandler
6. Run tests, build, type-check to verify
