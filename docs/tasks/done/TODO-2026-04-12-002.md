# Task: Playwright E2E UI Test Infrastructure

**Status:** completed
**Created:** 2026-04-12
**Completed:** 2026-04-12
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
The Intelli-Mock Web UI has two components implemented (mock-list and mock-detail) but no E2E UI tests. The design document (`docs/PLAYWRIGHT-E2E-DESIGN.md`) outlines the complete architecture. This task implements the Playwright infrastructure and basic E2E tests for the mock list component, following the design spec.

## Scope
### In Scope
- [ ] Install Playwright dependencies (`@playwright/test`, `playwright`)
- [ ] Create Playwright configuration (`test/playwright.config.ts`)
- [ ] Create test helpers (test-server, seed data)
- [ ] Create test setup fixtures (`test/setup.ts`)
- [ ] Add E2E test scripts to `package.json`
- [ ] Implement `test/e2e/mock-list.spec.ts` with 3 test scenarios:
  - Loading state (spinner visible)
  - Populated list (3 mock endpoints rendered)
  - Empty state (no mocks message)
- [ ] Validate tests pass with `pnpm test:e2e`

### Out of Scope
- Mock detail/component tests (will be added when more components are built)
- Full end-to-end intercept tests (requires mock handler + AI generation)
- CI/CD integration (Phase 7)
- Documentation updates (separate task)

## Steps
1. Install Playwright dependencies in `@intelli-mock/ui` package
2. Create `test/playwright.config.ts` with Chromium-only config, webServer setup
3. Create test helpers:
   - `test/helpers/test-server.ts` — Express server with sql.js, auth disabled
   - `test/helpers/seed.ts` — Database seeding utilities
4. Create `test/setup.ts` with Playwright fixtures (serverPort, authToken)
5. Add scripts to UI `package.json`: `test:e2e`, `test:e2e:ui`, `test:e2e:debug`
6. Implement `test/e2e/mock-list.spec.ts` with 3 test scenarios
7. Run `pnpm test:e2e` to validate all tests pass
8. Fix any issues (shadow DOM selectors, timing, server startup)

## Acceptance Criteria
- [ ] `pnpm --filter @intelli-mock/ui test:e2e` passes with 3/3 tests
- [ ] Tests run in Chromium browser with shadow DOM support
- [ ] Server starts automatically via Playwright's `webServer` config
- [ ] Database is seeded before each test
- [ ] Tests are deterministic (no flaky timeouts)
- [ ] Auth is disabled for E2E tests (`AUTH_DISABLED=true`)
- [ ] Test execution completes in < 30 seconds
