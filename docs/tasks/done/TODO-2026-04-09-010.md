# Task: Traffic Service Enhancement + Tests

**Status:** completed
**Created:** 2026-04-09
**Completed:** 2026-04-09
**Priority:** medium
**Estimated Time:** 20-30 mins

## Context
The `TrafficService` currently only has a `logTraffic()` method — it's write-only with no retrieval APIs. The PRD requires a traffic log viewer with 1-month retention (Functional requirement #19). Before Phase 3 (AI Engine), we need query methods to retrieve and inspect logged traffic, plus unit tests for the TrafficService which currently has zero test coverage.

This task adds retrieval methods to TrafficService and comprehensive unit tests.

## Scope
### In Scope
- [ ] `findAll(tenantId, options)` — list traffic logs with pagination (limit/offset)
- [ ] `findOne(id, tenantId)` — get single traffic log by ID, tenant-scoped
- [ ] `findByEndpoint(endpointId, tenantId, options)` — filter by mock endpoint
- [ ] `countByTenant(tenantId)` — total count for pagination
- [ ] Unit tests for all TrafficService methods (mocked TypeORM repository)

### Out of Scope
- Traffic log retention cron job (Phase 7)
- UI traffic log viewer component (Phase 6)
- Traffic controller/routes (will be added when retrieval is needed)

## Steps
1. **Enhance `TrafficService`** (`packages/intelli-mock-core/src/modules/mock/traffic.service.ts`)
   - Add `findAll(tenantId, { limit, offset })` — returns paginated list, ordered by created_at DESC
   - Add `findOne(id, tenantId)` — returns single TrafficLog or null
   - Add `findByEndpoint(endpointId, tenantId, { limit, offset })` — filter by endpoint, tenant-scoped
   - Add `countByTenant(tenantId)` — returns total count for pagination metadata
   - All queries scoped by `tenantId` via TrafficLog.tenantId relation through MockEndpoint

2. **Write unit tests** (`packages/intelli-mock-core/test/modules/mock/traffic.service.test.ts`)
   - Mock TypeORM TrafficLog repository with `vi.fn()`
   - Test `findAll` with pagination parameters
   - Test `findOne` — found and not found cases
   - Test `findByEndpoint` — filtering by endpoint ID
   - Test `countByTenant` — returns correct count
   - Test tenant isolation — ensure queries include tenantId filter
   - Target: 10+ tests

3. **Validate** — run tests, check coverage, ensure build passes

## Acceptance Criteria

- [ ] All methods are tenant-scoped (include tenantId in queries)
