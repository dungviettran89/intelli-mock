# Task: Sample Management API

**Status:** completed
**Created:** 2026-04-09
**Completed:** 2026-04-09
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Phase 2 (Mock CRUD + Matching) is complete for the mock module, but the **Sample management API** is still missing. This is a prerequisite for Phase 3 (AI Engine) since AI script generation requires a minimum of 5 sample pairs. The SamplePair entity and Sample service/controller are planned in the architecture but not yet implemented.

This task implements the REST API for managing sample request/response pairs, scoped by tenant and linked to MockEndpoints.

## Scope
### In Scope
- [ ] `SampleService` — CRUD operations for SamplePair, tenant-scoped, linked to MockEndpoint
- [ ] `SampleController` — REST API handlers (list, get, create, update, delete)
- [ ] `Sample routes` — Express router mounted at `/api/samples`
- [ ] Unit tests for SampleService (mocked TypeORM repository)
- [ ] Register sample module in DI container

### Out of Scope
- UI for sample management (Phase 6)
- AI script generation (Phase 3) — this task provides the foundation for it
- Bulk import/export of samples

## Steps
1. **Create `SampleService`** (`packages/intelli-mock-core/src/modules/sample/sample.service.ts`)
   - Inject SamplePair repository via tsyringe
   - Methods: `findAll(tenantId)`, `findOne(id, tenantId)`, `create(data, tenantId)`, `update(id, data, tenantId)`, `remove(id, tenantId)`, `countByEndpoint(endpointId, tenantId)`
   - All queries scoped by `tenantId`
   - Validate `endpointId` belongs to tenant before create

2. **Create `SampleController`** (`packages/intelli-mock-core/src/modules/sample/sample.controller.ts`)
   - Express request handlers mapping to SampleService
   - Routes: `GET /`, `GET /:id`, `POST /`, `PUT /:id`, `DELETE /:id`
   - Extract `tenantId` from `req.tenantId` (set by JWT middleware)
   - Return proper HTTP status codes and JSON responses

3. **Create routes** (`packages/intelli-mock-core/src/modules/sample/sample.routes.ts`)
   - Express Router with JWT auth middleware
   - Mount controller methods on routes
   - Export configured router

4. **Register in DI container** (`packages/intelli-mock-core/src/container.ts`)
   - Register SampleService and SampleController as singletons
   - Ensure SamplePair entity is included in TypeORM connection

5. **Write unit tests** (`packages/intelli-mock-core/test/modules/sample/sample.service.test.ts`)
   - Mock TypeORM repository with `vi.fn()`
   - Test all service methods with tenant scoping
   - Test error cases (not found, unauthorized, invalid endpoint)
   - Target: 12+ tests
