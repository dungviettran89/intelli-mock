**Completed:** 2026-04-09
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Phase 1 Foundation is 85% complete (tasks 001 and 002). The entities, TypeORM dual-driver support, DI container, and JWT auth middleware are all implemented. However, three critical pieces are missing before we can move to Phase 2 (Mock CRUD):

1. **Database migrations** — Required for production MariaDB deployments and proper schema versioning
2. **Server runner** (`server.ts`) — The app factory exists but nothing starts the HTTP server
3. **Error-handling middleware** — Global exception handler and CORS setup

Without these, the CLI app (`intelli-mock start`) has no way to actually start the server, and there's no migration path for production databases.

## Scope

- [ ] Create initial TypeORM migration (`packages/intelli-mock-core/src/database/migrations/`)
- [ ] Create `packages/intelli-mock-core/src/server.ts` — server runner that calls `createApp()` and listens on PORT
- [ ] Add global error-handling middleware to `app.ts`
- [ ] Add CORS middleware to `app.ts`
- [ ] Update `packages/intelli-mock-core/src/index.ts` to export server module
- [ ] Add TypeORM CLI config for running migrations

### Out of Scope

- CLI `start` / `init` commands (will be done separately)

## Steps
1. Create `database/migrations/InitialSchema.ts` — TypeORM migration that creates all 6 tables with proper constraints, indexes, and foreign keys
   - Use `MigrationInterface` from TypeORM
   - Implement `up()` and `down()` methods
   - Support both sql.js and MariaDB drivers (use cross-driver compatible SQL)

2. Create `server.ts` — Express server runner
   - Import `createApp()` from `app.ts`
   - Read PORT from env (default 3000)
   - Call `createApp()`, then `app.listen(PORT)`
   - Add graceful shutdown handlers (close DB connection on SIGTERM/SIGINT)
   - Export `startServer()` function for programmatic use

3. Update `app.ts` — Add middleware chain
   - Add CORS middleware (read `CORS_ORIGINS` from env, support multiple origins)
   - Add global error-handling middleware at the end of the chain (catch unhandled errors, return 500)
   - Ensure middleware order: CORS → body parsers → auth → routes → error handler

4. Create `database/data-source.config.ts` — TypeORM CLI configuration
   - Export `DataSource` instance for `typeorm` CLI commands
   - Read migration path from config

5. Update `package.json` scripts in `@intelli-mock/core`:
   - Add `migration:generate` script
   - Add `migration:run` script
   - Add `migration:revert` script

6. Update `index.ts` exports to include server module

7. Run `pnpm -r build` and `pnpm -r typecheck` to verify

## Acceptance Criteria
- [ ] `pnpm -r build` compiles without errors

- [ ] `server.ts` starts the Express server and listens on configured PORT

- [ ] CORS middleware correctly handles preflight requests and allowed origins
- [ ] Global error handler catches unhandled exceptions and returns 500 with error message
- [ ] TypeORM CLI can run migrations (`pnpm typeorm migration:run`)
