# Task: Phase 5 — Proxy Module

**Status:** completed
**Created:** 2026-04-11
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Phase 5 implements the Proxy Module, which enables Intelli-Mock to forward requests to real APIs and capture traffic automatically. This is a core feature that allows developers to proxy through existing endpoints, capture traffic, and fall back to AI-generated mocks when the upstream is unavailable. This phase follows Phase 4 (vm2 Sandbox) which is now complete.

## Scope
### In Scope
- [x] Proxy service with HTTP forwarding, timeout handling, and error management
- [x] Automatic traffic capture and logging to TrafficLog entity
- [x] Integration with existing mock handler for fallback logic
- [x] Unit tests for proxy service (HTTP fully mocked)

### Out of Scope
- Auto-endpoint route registration (`/_it/auto/*`) — will be done in a separate task
- Proxy configuration UI — belongs to Phase 6 (Web UI)
- Advanced retry logic or circuit breaker patterns

## Steps
1. Create `ProxyService` class with `forwardRequest()` method using Node's built-in `http`/`https` modules
2. Implement configurable timeout (default 30s), error handling, and response parsing
3. Add automatic traffic logging — log both request and response to `TrafficLog` entity
4. Create comprehensive unit tests with mocked HTTP requests (using `vi.mock()` for `http`/`https`)
5. Integrate proxy service into the DI container (`container.ts`)
6. Update `MockHandler` to support proxy fallback logic (optional, can be deferred)
7. Run tests, build, type-check, and format

## Acceptance Criteria
- [x] `ProxyService` class exists with `forwardRequest(url, requestOptions)` method
- [x] Configurable timeout via `PROXY_TIMEOUT` env var (default 30000ms)
- [x] Request/response pairs are logged to `TrafficLog` with `source: 'proxy'`
- [x] Network errors are caught and returned as structured error responses
- [x] Unit tests achieve 80%+ coverage on proxy service
