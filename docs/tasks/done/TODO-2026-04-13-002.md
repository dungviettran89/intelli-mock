# Task: Traffic API Integration Test

**Status:** completed
**Created:** 2026-04-13
**Priority:** medium
**Estimated Time:** 20-30 mins

## Context
The Traffic API (`/api/traffic`) is implemented with a controller, service, and routes, but only has unit tests (`traffic.controller.test.ts`, `traffic.routes.test.ts`). There is no integration test that validates the full HTTP pipeline — from Express router through controller to service layer — ensuring the traffic log retrieval, filtering, and deletion endpoints work end-to-end with a real (in-memory) database.

This task adds an integration test file for the Traffic API to close the testing gap, following the same pattern as existing integration tests (`mock.integration.test.ts`, `sample.integration.test.ts`).

## Scope
### In Scope
- [ ] Create `test/modules/traffic/traffic.integration.test.ts` with full HTTP-level integration tests
- [ ] Test `GET /api/traffic` — retrieves traffic logs with tenant scoping
- [ ] Test `DELETE /api/traffic/:id` — deletes a traffic log entry
- [ ] Test `DELETE /api/traffic/older-than/:days` — bulk deletion by age
- [ ] Use real in-memory sql.js database (no mocking of service layer)
- [ ] Maintain 100% offline — no external dependencies

### Out of Scope
- UI component tests for traffic-viewer (Phase 6)
- Proxy traffic capture integration (already covered in proxy/auto handler tests)
- Performance/load testing

## Steps
1. Read existing `test/modules/traffic/traffic.routes.test.ts` and `test/modules/mock/traffic.service.test.ts` to understand the API contract
2. Read existing integration test patterns (e.g., `test/modules/sample/sample.integration.test.ts`) for conventions
3. Create `test/modules/traffic/traffic.integration.test.ts` with:
   - Test setup: create tenant, mock endpoint, seed traffic logs via TypeORM
   - `GET /api/traffic` — returns paginated traffic logs, filtered by tenant
   - `GET /api/traffic?limit=5` — respects query parameters
   - `DELETE /api/traffic/:id` — deletes specific log, returns 204
   - `DELETE /api/traffic/:id` — returns 404 for non-existent log
   - `DELETE /api/traffic/older-than/7` — bulk deletes logs older than N days
4. Run tests to verify they pass with exit code 0
5. Run full test suite to ensure no regressions
