# Task: Wire AI Script Generation to REST API

**Status:** completed
**Completed:** 2026-04-10
**Created:** 2026-04-10
**Priority:** high
**Estimated Time:** 20-30 mins

## Context

Phase 3 source files (AIService, ScriptService, ScriptValidator, prompts) are all implemented with comprehensive unit tests (47 tests). However, the AI script generation pipeline is **not wired to the REST API**. The PRD specifies `POST /api/mocks/:id/generate` should orchestrate: sample count check → AI generation → validation → script creation → return code.

## Scope

### In Scope
- [ ] Add `generate` method to `MockController` that orchestrates AI script generation
- [ ] Add `POST /api/mocks/:id/generate` route in `mock.routes.ts`
- [ ] Wire AIService, ScriptService, SampleService, ScriptValidator into MockController via DI
- [ ] Implement sample count >= 5 validation with proper 503 response
- [ ] Return generated script with metadata (code, version, model, token usage)
- [ ] Unit tests for the generate endpoint (6-8 tests)

### Out of Scope
- AI service implementation (already exists)
- Script service implementation (already exists)
- Script validator implementation (already exists)
- Sample service implementation (already exists)
- UI integration (Phase 6)

## Steps

1. **Update MockController** — inject AIService, ScriptService, SampleService; add `generate()` method
2. **Update mock.routes.ts** — add `POST /:id/generate` route
3. **Write unit tests** — test the generate endpoint with mocked services
4. **Run full test suite** — ensure all 220+ tests pass
5. **Build and type-check** — verify compilation

## Acceptance Criteria

- [x] `POST /api/mocks/:id/generate` returns 503 when sample count < 5
