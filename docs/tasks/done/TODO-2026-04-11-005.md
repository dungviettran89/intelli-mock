# Task: CLI Application ŌĆö Start Command with Server + UI

**Status:** completed
**Created:** 2026-04-11
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
The CLI application at `apps/intelli-mock/` is currently scaffolded with only a basic Commander.js setup. To make the application launchable, we need to implement the `start` command that boots the Express server and serves the UI. This is a critical milestone ŌĆö without it, users cannot run the application end-to-end.

## Scope
### In Scope
- [ ] Implement `start` command with all auth flags (`--no-auth`, `--auth-key`, `--auth-issuer`, `--auth-algorithm`, `--port`)
- [ ] Implement config loader that merges CLI flags > env vars > defaults
- [ ] Wire `start` command to call `startServer()` from `@intelli-mock/core`
- [ ] Serve static UI files from `@intelli-mock/ui/dist` via Express
- [ ] Add console output showing server URL and UI URL
- [ ] Handle graceful shutdown (already implemented in core, just needs to be called)

### Out of Scope
- `init` command (will be separate task)
- Config file loading (YAML/JSON) ŌĆö env vars and CLI flags only for now
- CLI unit tests (will be separate task)
- Auto-building UI before serving ŌĆö assume `pnpm -r build` was run

## Steps
1. Create `apps/intelli-mock/src/config.ts` ŌĆö config loader that merges CLI flags > env vars > defaults
2. Create `apps/intelli-mock/src/commands/start.ts` ŌĆö start command implementation with all flags
3. Update `apps/intelli-mock/src/cli.ts` ŌĆö wire up the start command to the CLI
4. Ensure UI static files are served ŌĆö add Express static middleware to serve UI from core package
5. Test by running `pnpm -r build && node apps/intelli-mock/dist/cli.js start --no-auth`
6. Verify server starts on correct port and UI is accessible

## Acceptance Criteria
- [x] `intelli-mock start --no-auth` launches server on default port 3000
- [x] `intelli-mock start --port 4000 --no-auth` launches on port 4000
- [x] Auth flags properly override env vars (`--auth-key`, `--auth-issuer`, `--auth-algorithm`)
- [x] UI is accessible at `http://localhost:{port}/`
- [x] API routes accessible at `http://localhost:{port}/api/*`
- [x] Mock runtime routes accessible at `http://localhost:{port}/_it/*`
- [x] Console output shows URLs: server port, API docs, UI
- [x] Graceful shutdown on Ctrl+C (SIGINT/SIGTERM)
- [x] All packages build without errors (`pnpm -r build`)
- [x] Type-checking passes (`pnpm -r typecheck`)
