# Task: Test/Try Endpoint for Mock Scripts

**Status:** completed
**Created:** 2026-04-11
**Completed:** 2026-04-11
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Phase 4 (vm2 Sandbox) is marked complete in PROGRESS.md, but the Test/Try endpoint is explicitly listed as "Not Started". This endpoint allows users to test a mock script without activating it, which is critical for the Web UI's "Try-it panel" in Phase 6. Without this, there's no way to validate a script before deployment.

## Scope
### In Scope
- [ ] Create `POST /api/scripts/:id/test` endpoint in ScriptController
- [ ] Accept request body with method, params, query, headers, body (same as mock runtime context)
- [ ] Execute the script using ScriptRunner in a sandboxed context
- [ ] Return structured result: `{ success, response?, error?, executionTimeMs }`
- [ ] Add unit tests for ScriptController test endpoint (8-10 tests)
- [ ] Wire up routes in app.ts

### Out of Scope
- UI integration (Phase 6)
- Traffic logging for test requests (test endpoint shouldn't pollute traffic logs)
- Script validation/syntax checking (already exists in ScriptValidator)

## Steps
1. **Create test endpoint in ScriptController**
   - Add `test()` method to `ScriptController` in `src/modules/script/script.controller.ts` (create if doesn't exist)
   - Extract request context from body: `{ method, params?, query?, headers?, body? }`
   - Fetch the MockScript by ID
   - Call `ScriptRunner.run()` with the script and context
   - Return structured result (200 for success, 404 if script not found, 400 if invalid input)

2. **Create ScriptController and routes** (if not exists)
   - Create `src/modules/script/script.controller.ts`
   - Create `src/modules/script/script.routes.ts` with `createScriptRouter()`
   - Wire up in `src/app.ts` under `/api/scripts`

3. **Write comprehensive unit tests**
   - Create `test/modules/script/script.controller.test.ts`
   - Test cases:
     - ✅ Successful script execution returns result
     - ✅ Script not found returns 404
     - ✅ Missing method field returns 400
     - ✅ Script with validation error returns error result
     - ✅ Script execution timeout returns timeout error
     - ✅ Script runtime error returns runtime error
     - ✅ Custom headers and body passed to script
     - ✅ URL params extracted correctly
   - Use fully mocked DI container, ScriptRunner, and ScriptService

4. **Wire up routes in app.ts**
   - Import `createScriptRouter` and mount under `/api/scripts`
   - Ensure auth middleware protects the endpoint

5. **Run tests and validate**
   - Run `pnpm test` in core package
   - Ensure all existing tests still pass
   - Verify coverage hasn't dropped

## Acceptance Criteria
