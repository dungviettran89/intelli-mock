# Task: Traffic Log API (`/api/traffic`)

**Status:** completed
**Created:** 2026-04-12
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
The `TrafficService` is fully implemented at `modules/mock/traffic.service.ts` with list/query methods. The `TrafficLog` entity is defined. The service is registered in the DI container and exported from `index.ts`. However, there is no REST API endpoint to expose traffic logs to the UI or external consumers. This is a clear gap — the architecture specifies a `/api/traffic` route that doesn't exist.

## Scope
### In Scope
- [ ] Create `modules/traffic/traffic.controller.ts` with list endpoint (tenant-scoped)
- [ ] Create `modules/traffic/traffic.routes.ts` with Express Router wiring
- [ ] Register `/api/traffic` route in `app.ts`
- [ ] Export types from `index.ts`
- [ ] Unit tests for controller (mocked service)

### Out of Scope
- Traffic log retention cron (Phase 7, larger effort)
- Swagger/OpenAPI docs (Phase 7)
- WebSocket support (not yet scoped)

## Steps
1. Create `packages/intelli-mock-core/src/modules/traffic/traffic.controller.ts` — inject `TrafficService`, expose `GET /api/traffic` returning paginated logs scoped by tenant
2. Create `packages/intelli-mock-core/src/modules/traffic/traffic.routes.ts` — Express Router with `createTrafficRouter()` function, wiring controller to routes
3. Add `app.use('/api/traffic', createTrafficRouter(container))` to `app.ts`
4. Export controller + routes from `packages/intelli-mock-core/src/index.ts`
5. Create `test/modules/traffic/traffic.controller.test.ts` — unit tests with mocked service
6. Build and test

## Acceptance Criteria
- [ ] `GET /api/traffic` returns traffic logs for the authenticated tenant
- [ ] Response includes `route`, `method`, `path`, `request`, `response`, `source`, `createdAt` fields
- [ ] Tenant isolation enforced (only current tenant's logs returned)
- [ ] `pnpm -r build` passes
- [ ] TypeScript strict mode passes
- [ ] Unit tests pass for new controller
