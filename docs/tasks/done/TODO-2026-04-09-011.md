# Task: Script Service + Validator

**Status:** pending
**Created:** 2026-04-09
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Phase 2 (Mock CRUD + Matching) is complete. The next phase is Phase 3: AI Engine. Before the AI service can generate scripts, we need a **Script Service** to manage script lifecycle (create, version, activate) and a **Script Validator** to check syntax before saving. The MockScript entity already exists but has no service layer.

This task implements the foundation for AI-powered script generation — scripts must be versioned (only one active per endpoint) and validated before storage.

## Scope
### In Scope
- [ ] `ScriptValidator` — syntax check for generated JavaScript (try `new Function()` or `vm.Script`)
- [ ] `ScriptService` — CRUD + versioning for MockScript, tenant-scoped via endpoint
- [ ] `activate(id, tenantId)` — activate a script (deactivates all others for same endpoint)
- [ ] `findActiveByEndpoint(endpointId, tenantId)` — get the currently active script
- [ ] Unit tests for ScriptValidator (valid/invalid code, edge cases)
- [ ] Unit tests for ScriptService (mocked TypeORM)

### Out of Scope
- AI service integration (Vercel AI SDK) — next task
- vm2 sandbox execution (Phase 4)
- Script controller/routes — will be added with AI generation endpoint

## Steps
1. **Create `ScriptValidator`** (`src/modules/script/script.validator.ts`)
   - `validate(code: string): { valid: boolean, error?: string }`
   - Use `new Function(code)` for basic syntax check (doesn't execute)
   - Handle edge cases: empty code, non-string input

2. **Create `ScriptService`** (`src/modules/script/script.service.ts`)
   - Inject MockScript repository
   - Methods: `create(endpointId, code, aiModel, aiPrompt)`, `findAll(endpointId, tenantId)`, `findOne(id, tenantId)`, `findActive(endpointId)`, `activate(id, tenantId)`, `countByEndpoint(endpointId, tenantId)`
   - On `create`: auto-increment version, set isActive=false
   - On `activate`: deactivate all scripts for endpoint, set target isActive=true
   - All queries scoped by verifying endpoint belongs to tenant

3. **Register in DI container** (`container.ts`)

4. **Write unit tests** (`test/modules/script/script.validator.test.ts` + `test/modules/script/script.service.test.ts`)
   - Validator: valid code, syntax errors, empty input, complex valid code
   - Service: create with versioning, activate (transactional), find active, tenant isolation
   - Target: 15+ tests total
