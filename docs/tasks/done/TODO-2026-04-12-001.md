# Task: Phase 6 — Mock Detail View Component

**Status:** completed
**Created:** 2026-04-12
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
The Phase 6 Web UI currently has only the `mock-list` component. The next logical component is the **Mock Detail View**, which displays a single mock endpoint's full information including path pattern, method, status, proxy URL, prompt extra, and associated samples/scripts. This component is a prerequisite for the Script Editor and Sample Management UI components.

## Scope
### In Scope
- [ ] Create `mock-detail.ts` Lit Element component
- [ ] Display full mock endpoint details (path, method, status, proxy URL, priority, prompt extra)
- [ ] Add loading, error, and empty states
- [ ] Add "Generate Script" button (wired to API)
- [ ] Add navigation back to list
- [ ] Add status badge and method badge (reuse styles from mock-list)
- [ ] Update `index.ts` to export the new component
- [ ] Update `PROGRESS.md`

### Out of Scope
- Script editor (CodeMirror 6) — separate component
- Sample management UI — separate component
- Try-it panel — separate component
- Playwright E2E tests — separate task

## Steps
1. Read existing `mock-list.ts` for style conventions and patterns
2. Read API service (`api.ts`) to understand data shapes and available endpoints
3. Create `packages/intelli-mock-ui/src/components/mock-detail.ts` with:
   - `@state()` for mock data, loading, error
   - `@property()` for mock ID (set via URL or attribute)
   - Fetch mock detail on connect
   - Render method badge, status badge, proxy URL, priority, prompt extra
   - Generate script button calling `POST /api/mocks/:id/generate`
   - Back button emitting custom event
4. Update `packages/intelli-mock-ui/src/index.ts` to import and register the component
5. Update `docs/PROGRESS.md` to mark Mock Detail as done
6. Verify TypeScript compilation: `pnpm --filter @intelli-mock/ui build`
