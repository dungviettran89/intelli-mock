# Task: Traffic Log Retention Cron — TrafficLogger + RetentionCron Services

**Status:** completed
**Created:** 2026-04-12
**Priority:** high
**Estimated Time:** 20-30 mins

## Context

Phase 7: Polish requires a traffic log retention mechanism. The `TrafficLog` entity has a `createdAt` timestamp, and the PRD specifies a **1-month retention policy**. Currently, traffic logs accumulate indefinitely with no cleanup. This task implements the `TrafficLogger` service (logging abstraction) and `RetentionCron` service (scheduled deletion of logs older than 30 days), plus wires them into the server lifecycle.

## Scope

### In Scope
- [x] Create `src/core/logging/traffic-logger.ts` — `TrafficLogger` service with `log()` method that delegates to `TrafficService.logTraffic()`
- [x] Create `src/core/logging/retention-cron.ts` — `RetentionCron` service with `start()`/`stop()` methods, runs daily, deletes logs older than 30 days
- [x] Register both services in `src/container.ts`
- [x] Wire `RetentionCron.start()` into `startServer()` and `RetentionCron.stop()` into `stopServer()`
