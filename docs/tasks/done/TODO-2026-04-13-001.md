# Task: Add Missing Entity Fixtures + Sample Delete Route + Integration Tests

**Status:** completed
**Created:** 2026-04-13
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
The test helper `test/helpers/fixtures.ts` only provides factory functions for 3 entities (Tenant, MockEndpoint, User), but is missing factories for SamplePair, MockScript, and TrafficLog. This forces test writers to manually construct these entities, leading to duplication and inconsistencies.

Additionally, the Sample routes only register `POST /api/samples` but the controller has a `deleteSample` method that is not wired to a route. The API spec in `docs/API.md` documents `DELETE /api/mocks/:id/samples/:sampleId`.

This task adds the missing entity factories and wires up the sample delete route, then adds integration tests to validate the full pipeline.

## Scope
### In Scope
- [ ] Add factory functions for SamplePair, MockScript, TrafficLog to `test/helpers/fixtures.ts`
- [ ] Add `DELETE /api/mocks/:id/samples/:sampleId` route to `src/modules/sample/sample.routes.ts`
- [ ] Add `GET /api/mocks/:id/samples` route to list samples for an endpoint
- [ ] Create integration tests for sample delete and list endpoints
- [ ] Update existing tests that manually construct these entities to use the new factories

### Out of Scope
- UI component tests for sample editor (already exists)
- Proxy or AI-related integration tests
- Swagger/OpenAPI documentation

## Steps
1. Read `test/helpers/fixtures.ts` to understand existing factory patterns
2. Read `src/entities/sample-pair.entity.ts`, `src/entities/mock-script.entity.ts`, `src/entities/traffic-log.entity.ts` to understand entity structures
3. Add factory functions to `fixtures.ts`:
   - `createSamplePair(overrides?)` — creates a SamplePair with defaults
   - `createMockScript(overrides?)` — creates a MockScript with defaults
   - `createTrafficLog(overrides?)` — creates a TrafficLog with defaults
4. Read `src/modules/sample/sample.routes.ts` and `src/modules/sample/sample.controller.ts`
5. Add missing routes:
   - `DELETE /api/mocks/:id/samples/:sampleId` → `sampleController.deleteSample`
   - `GET /api/mocks/:id/samples` → `sampleController.getSamples`
6. Read existing integration test patterns (`test/modules/sample/sample.integration.test.ts`)
7. Create `test/modules/sample/sample.delete-integration.test.ts` with:
   - Test setup: create tenant, mock endpoint, seed samples using new factories
   - `DELETE /api/mocks/:id/samples/:sampleId` — deletes a sample, returns 204
   - `DELETE /api/mocks/:id/samples/:sampleId` — returns 404 for non-existent sample
   - `DELETE /api/mocks/:id/samples/:sampleId` — returns 403 for cross-tenant attempt
   - `GET /api/mocks/:id/samples` — lists all samples for an endpoint
   - `GET /api/mocks/:id/samples` — returns empty array if no samples
8. Run tests to verify they pass with exit code 0
9. Run full test suite to ensure no regressions
