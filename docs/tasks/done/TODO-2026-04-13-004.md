# Task: Swagger/OpenAPI Documentation

**Status:** completed
**Created:** 2026-04-13
**Completed:** 2026-04-13
**Priority:** high
**Estimated Time:** 20-30 mins

## Context
Swagger/OpenAPI documentation is a core functional requirement in the PRD (`docs/PRD.md` - Functional requirement #18). The server currently logs a URL for `/api-docs` but no actual documentation is served. This feature will provide interactive API documentation for developers using the platform.

## Scope
### In Scope
- [ ] Install `swagger-ui-express` and `swagger-jsdoc` dependencies
- [ ] Create OpenAPI 3.0 specification file with JSDoc annotations
- [ ] Document all existing API routes:
  - Mock endpoints CRUD (`/api/mocks`)
  - Sample pairs management (`/api/samples`)
  - Script operations (`/api/scripts`)
  - Traffic logs (`/api/traffic`)
  - Mock runtime (`/_it/mock/*`)
  - Auto endpoint (`/_it/auto/*`)
- [ ] Mount Swagger UI at `/api-docs` in Express app factory
- [ ] Add unit tests for Swagger setup

### Out of Scope
- API versioning
- Authentication schemes in Swagger (JWT Bearer can be added later)
- Custom Swagger themes or branding

## Steps
1. Install dependencies: `swagger-ui-express` and `swagger-jsdoc` in `@intelli-mock/core`
2. Create `src/docs/openapi.ts` with OpenAPI 3.0 spec:
   - Info, servers, tags
   - Path definitions for all API routes
   - Request/response schemas for each entity
   - JSDoc comments on controllers
3. Update `src/app.ts` to mount Swagger UI at `/api-docs`
4. Create `test/docs/openapi.test.ts` with unit tests:
   - Verify `/api-docs` endpoint returns HTML
   - Verify OpenAPI spec is valid JSON
   - Verify all routes are documented
5. Run tests and build to verify no regressions
6. Update `docs/PROGRESS.md` to mark this item as complete
